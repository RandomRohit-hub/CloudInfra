# VProfile Application Troubleshooting Documentation

## Issue: VProfile Application Not Opening on Tomcat

### Environment

| Component          | Technology       |
| ------------------ | ---------------- |
| Application Server | Apache Tomcat 10 |
| Application        | VProfile         |
| Database           | MySQL            |
| Cache              | Memcached        |
| Message Broker     | RabbitMQ         |
| DNS                | Route 53         |
| OS                 | Ubuntu 24.04     |
| Cloud              | AWS EC2          |

---

# Problem Statement

When accessing:

```text
http://98.93.29.106:8080/
```

The browser displayed:

```text
HTTP Status 404 – Not Found
Apache Tomcat/10.1.16
```

The VProfile application was not loading.

---

# Initial Assumption

At first glance, it appeared that:

* Tomcat was not running
* Application deployment failed
* Security Group issue
* Port 8080 issue

However, these assumptions were incorrect.

---

# Troubleshooting Process

## Step 1: Verify Tomcat Status

Command:

```bash
sudo systemctl status tomcat10
```

Result:

```text
Tomcat Active (Running)
```

### Conclusion

Tomcat service was healthy.

---

## Step 2: Verify Application Deployment

Command:

```bash
sudo ls /var/lib/tomcat10/webapps
```

Output:

```text
ROOT
ROOT.war
```

### Conclusion

Application WAR file was successfully deployed.

---

## Step 3: Verify Application Files

Command:

```bash
sudo find /var/lib/tomcat10/webapps/ROOT -name "*.properties"
```

Output:

```text
application.properties
validation.properties
```

### Conclusion

Application extracted successfully.

---

## Step 4: Check Tomcat Logs

Command:

```bash
sudo tail -100 /var/log/tomcat10/catalina.out
```

Important Error:

```text
AuthenticationFailureException

ACCESS_REFUSED

Login was refused using authentication mechanism PLAIN
```

### Observation

The application startup was failing while connecting to RabbitMQ.

---

# Root Cause Analysis

## Check RabbitMQ Configuration

Application Configuration:

```bash
sudo cat \
/var/lib/tomcat10/webapps/ROOT/WEB-INF/classes/application.properties
```

Output:

```properties
rabbitmq.address=rmq01.vpofile.in
rabbitmq.port=5672
rabbitmq.username=rabbit
rabbitmq.password=bunny
```

Application expected:

```text
Username = rabbit
Password = bunny
```

---

## Check RabbitMQ Users

On RabbitMQ Server:

```bash
sudo rabbitmqctl list_users
```

Output:

```text
guest
test
```

### Problem Found

RabbitMQ did not contain:

```text
rabbit
```

user.

---

# Why Application Failed

Startup sequence:

```text
Tomcat
   ↓
Load VProfile
   ↓
Connect MySQL
   ↓
Connect Memcached
   ↓
Connect RabbitMQ
```

RabbitMQ connection:

```text
Username = rabbit
Password = bunny
```

RabbitMQ response:

```text
ACCESS_REFUSED
```

Authentication failed.

---

# Why Browser Showed 404

Many people assume:

```text
404 = Tomcat Problem
```

Not true.

Actual flow:

```text
Tomcat Running
      ↓
Application Starts
      ↓
RabbitMQ Authentication Fails
      ↓
Spring Context Stops
      ↓
Application Unavailable
      ↓
Browser Receives 404
```

Tomcat was working.

The application was not.

---

# Solution

Create the user expected by the application.

On RabbitMQ Server:

```bash
sudo rabbitmqctl add_user rabbit bunny

sudo rabbitmqctl set_user_tags rabbit administrator

sudo rabbitmqctl set_permissions \
-p / rabbit ".*" ".*" ".*"
```

Verify:

```bash
sudo rabbitmqctl list_users
```

Expected:

```text
guest
test
rabbit
```

---

# Restart Application

On App Server:

```bash
sudo systemctl daemon-reload

sudo systemctl restart tomcat10
```

Verify:

```bash
sudo systemctl status tomcat10
```

---

# Verification

Check logs:

```bash
sudo tail -50 /var/log/tomcat10/catalina.out
```

Authentication error should disappear.

Open:

```text
http://98.93.29.106:8080/
```

Application should load successfully.

---

# Architecture Dependency Learning

VProfile depends on multiple services:

```text
User
 ↓
Tomcat
 ├── MySQL
 ├── Memcached
 └── RabbitMQ
```

If any dependency fails:

```text
Database ❌
Cache ❌
RabbitMQ ❌
```

Application startup may fail.

---

# Key Learning Points

### 1. Always Check Logs

Never assume the issue from the browser error.

```text
Browser Error
     ≠
Root Cause
```

Logs reveal the actual problem.

---

### 2. Verify Dependencies

For Java applications verify:

```text
Database
Message Queue
Cache
DNS
Network Connectivity
```

before investigating Tomcat.

---

### 3. 404 Does Not Always Mean Missing File

Possible causes:

```text
Application not deployed
Application crashed
Dependency failure
Wrong context path
```

---

### 4. Read application.properties

This file contains:

```text
Database host
Database port
RabbitMQ host
RabbitMQ credentials
Memcache host
```

Useful for troubleshooting.

---

# DevOps Troubleshooting Flow

```text
Application Not Opening
         ↓
Check Security Group
         ↓
Check Port
         ↓
Check Service Status
         ↓
Check Deployment
         ↓
Check Logs
         ↓
Identify Dependency Failure
         ↓
Fix Dependency
         ↓
Restart Service
         ↓
Verify Application
```

# Final Root Cause

**The VProfile application could not authenticate with RabbitMQ because the application expected the RabbitMQ user `rabbit` with password `bunny`, but that user did not exist on the RabbitMQ server. Tomcat was healthy, the WAR file was deployed correctly, but the application startup failed due to RabbitMQ authentication failure, resulting in a 404 page in the browser.**
